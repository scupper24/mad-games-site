# БРИФ: Сайт игровой студии MAD GAMES

---

## 1. О проекте

**Название студии:** MAD GAMES  
**Тип проекта:** Корпоративный сайт-визитка игровой студии  
**Язык:** Русский (единственный, без мультиязычности)  
**Хостинг:** Локальный сервер (localhost). Деплой и домен — вне scope задачи.

---

## 2. Цели сайта

Сайт решает три задачи одновременно:

1. **Портфолио** — презентация выпущенных игр студии (5+ тайтлов)
2. **HR** — привлечение специалистов в команду через страницу вакансий
3. **Медиа / PR** — предоставление прессе информации о студии

---

## 3. Целевая аудитория

| Аудитория | Что ищут на сайте |
|---|---|
| Игроки / фанаты | Информация об играх, трейлеры, ссылки на площадки |
| Журналисты / медиа | Описание студии, факты о команде и играх |
| Соискатели | Открытые вакансии, форма отклика |

## 4. Структура сайта (страницы)

### 4.1 Главная (`/`)
Hero-секция с названием студии и коротким слоганом. Включает:
- Броский визуальный hero с анимацией
- Краткое описание студии (кто мы, что делаем)
- Превью последних / featured игр (карточки, 3–4 штуки) со ссылкой на полный каталог
- Блок «Присоединиться к команде» — CTA на страницу вакансий
- Ссылки на соцсети в футере и/или header

### 4.2 Игры / Портфолио (`/games`)
Каталог всех игр студии. Для каждой игры:
- Обложка / ключевой арт
- Название
- Краткое описание (жанр, платформа, год выпуска)
- Встроенный трейлер / видео (YouTube embed или `<video>`)
- Кнопка «Играть» / ссылка на магазин (браузер, Google Play, App Store)
- Скриншоты (галерея или слайдер)

> **Placeholder-контент:** пока игры заполняются моковыми данными (5–6 карточек). Структура данных должна быть вынесена в отдельный файл (`src/data/games.ts`), чтобы потом легко заменить на реальный контент.

### 4.3 Вакансии (`/jobs`)
Список открытых позиций. Для каждой вакансии:
- Название позиции
- Формат работы (удалённо / офис / гибрид)
- Краткое описание требований и задач
- Кнопка «Откликнуться» — открывает форму или mailto-ссылку

> **Placeholder-контент:** 3–4 макап-вакансии. Данные вынести в `src/data/jobs.ts`.

> **Форма отклика:** отправка через `mailto:Kurapov.99@yandex.ru` (без серверной логики, статика). Поля: имя, должность, ссылка на портфолио / резюме, сообщение.

## 5. Общий дизайн и стиль

### 5.1 Логотип
Логотип — текстовый, шрифт **Unbounded** (Google Fonts), начертание Bold или ExtraBold, текст `MAD GAMES`. Без дополнительных иконок на первом этапе.

### 5.2 Типографика
- **Display / заголовки:** Unbounded (уже используется для логотипа — сохраняет единство)
- **Body:** подобрать читаемый sans-serif с характером (не Inter, не Roboto). Варианты: Space Grotesk (слишком часто), **Syne**, **DM Sans**, **Figtree**, **Instrument Sans**

### 5.3 Цветовая палитра
Общее направление — уточнить по референсам, ориентир: **тёмная, насыщенная, с яркими акцентами** (типичная для инди/игровой эстетики).

Все цвета — через CSS Custom Properties:
```css
:root {
  --color-bg: ...;
  --color-surface: ...;
  --color-text: ...;
  --color-accent: ...;
  --color-accent-2: ...;
}
```

### 5.4 Настроение и эстетика
Студия делает **инди и казуальные игры для браузера и мобайла** — сайт должен отражать:
- Энергию, нестандартность, творческую свободу
- Не корпоративный стиль, но и не детская безделушка
- Ощущение «сделано с душой небольшой командой»

Избегать:
- Скучных корпоративных шаблонов
- Перегружённого «киберпанка» без смысла
- Стандартных фиолетовых градиентов на белом

### 5.5 Анимации и интерактивность
- Плавные переходы между страницами
- Staggered-анимация появления карточек игр
- Hover-эффекты на карточках (масштаб, свечение, parallax-лёгкий эффект)
- Hero-секция с анимированным элементом (частицы / шейдер / CSS-анимация)
- Никаких тяжёлых Three.js-сцен без необходимости — приоритет производительности

| Компонент | Решение |
|---|---|
| Фреймворк | **Astro** (статическая генерация) |

## 9. Требования к качеству кода

- Семантический HTML (nav, main, section, article, footer)
- Адаптивный дизайн: mobile-first, брейкпоинты 375 / 768 / 1280px
- Доступность: alt у изображений, aria-label у иконочных кнопок, фокус-стили
- Производительность: lazy loading изображений, минимум внешних зависимостей
- Чистый, читаемый код

## 10. Что НЕ входит в scope

- Серверная логика, бэкенд, база данных
- CMS или панель администратора
- Мультиязычность
- Покупки / монетизация внутри сайта
- SEO-оптимизация (можно заложить базовые мета-теги, но не приоритет)
- Деплой и настройка домена
